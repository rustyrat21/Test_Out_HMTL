# Get all apps installed on a workstations or server To pull a list of installed applications that are not installed by default on a Windows system using PowerShell, you can utilize the Windows Management Instrumentation (WMI) and filter out the default applications. Here's a PowerShell script to accomplish this:

Code Snippet


        # Define an array of default application names
$DefaultApps = @(
    "Microsoft Office",
    "Microsoft Visual C++",
    "Microsoft Silverlight",
    "Microsoft .NET Framework",
    "Microsoft Visual Studio",
    "Adobe Acrobat",
    "Java",
    "Google Chrome",
    "Mozilla Firefox",
    "Microsoft SQL Server",
    "Microsoft Teams",
    "Microsoft OneDrive",
    "Microsoft Edge",
    "Windows Live",
    "Windows SDK",
    "Microsoft Office Click-to-Run",
    "Microsoft Office Suite"
)

# Get a list of installed applications
$InstalledApps = Get-WmiObject -Query "SELECT * FROM Win32_Product" | Select-Object -Property Name

# Filter out default applications
$CustomApps = $InstalledApps | Where-Object { $DefaultApps -notcontains $_.Name }

# Display the list of custom applications
$CustomApps | Format-Table -AutoSize

Markdown Content

In this script: 1. We define an array `$DefaultApps` containing the names of default applications. You can customize this list with additional default application names if needed. 2. We use the `Get-WmiObject` cmdlet to retrieve a list of installed applications via WMI. We specifically query the `Win32_Product` class. 3. We filter the installed applications to exclude those with names found in the `$DefaultApps` array, leaving only custom applications. 4. Finally, we display the list of custom applications using `Format-Table`. Please note that querying the `Win32_Product` class can be relatively slow and resource-intensive. It may take some time to gather the list of installed applications. Additionally, you should run this script with appropriate permissions, as querying installed applications requires administrative access.
